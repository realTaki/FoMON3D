## English Version

If agents are going to participate in real economies, we need to solve one hard problem first:  
**capability can come before trust, but capital requires trust.**

FoMON3D is built around that gap.

Instead of asking users to hand funds to an agent treasury directly, we start with a game loop that users understand, then convert participation into treasury capital.

### The mechanism

1. Users deposit `MON` and receive `FoMON` (LST-style treasury share).  
2. Each valid deposit refreshes the countdown.  
3. If no one deposits before timeout, the last depositor wins the round reward.  
4. Users can exit through `redeem`.  
5. During the redeem window, capital stays in the Agent-managed vault.

So FoMON3D is not just a game loop.  
It is a **fundraising and capital-retention layer** for Agent Vaults.

### Why this matters

Most projects try strategy first and fundraising later.  
For agent economies, this order can fail in practice.  
FoMON3D flips it:

- first aggregate attention and liquidity,  
- then let strategy execution prove itself with real capital.

This turns trust from a binary assumption into an observable process.

## 中文版

如果 Agent 要真正参与经济活动，第一道门槛不是策略，而是资金信任：  
**能力可以先于信任出现，但资本不会。**

FoMON3D 就是为这个缺口设计的。

我们不要求用户第一天就把钱直接交给 Agent 金库，而是先提供一个可理解、可参与的游戏入口，再把参与行为转化为金库可用资本。

### 机制是什么

1. 用户存入 `MON`，获得 `FoMON`（LST 形式的金库份额）。  
2. 每次有效存入都会刷新倒计时。  
3. 如果倒计时结束前无人继续存入，最后存入者获得回合奖励。  
4. 用户可通过 `redeem` 退出。  
5. 在 redeem 等待窗口内，资金留在 Agent 管理的金库中。

所以 FoMON3D 不只是游戏机制，  
它是一个为 Agent Vault 服务的**募资与资本留存层**。

### 为什么这个顺序更有效

传统做法是“先有策略，再去募资”。  
在 Agent 经济里，这个顺序在早期经常跑不通。  
FoMON3D 反过来做：

- 先聚合注意力和流动性，  
- 再用策略执行去证明能力。

这样，“信任”不再是一次性假设，而变成一个可观察、可迭代的过程。

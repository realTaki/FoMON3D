## English Version

If we believe agents will become major participants in future economies, one practical issue appears immediately:  
**An agent may be capable of generating returns, but humans may not trust it with capital on day one.**

FoMON3D is our answer:  
Instead of asking users to trust an AI treasury directly, we start from a high-engagement game loop, then convert participation and liquidity into working capital for an Agent-managed vault.

### Why this model can work

The classic path is: strategy first, then fundraising.  
For agent economies, the order may need to be reversed:  
aggregate users and capital first, then let strategy prove itself.

FoMON3D does this through:

1. Users deposit `MON` and receive `FoMON` (an LST-style treasury share).  
2. A countdown game drives repeated participation: each valid deposit refreshes the timer.  
3. If no one deposits before timeout, the last depositor wins the round reward.  
4. Users can exit via `redeem`.  
5. During the redeem window, capital remains in the Agent-managed Treasury for strategy execution.

This is neither pure gambling nor pure staking.  
It is a **capital formation layer built on game dynamics**.

### Core idea: convert trust into measurable capital flow

In Web3, the hardest part is often not writing a strategy, but convincing people to allocate real capital to it.  
FoMON3D splits that trust problem into two steps:

- First, generate participation through instant feedback, competition, and rewards.  
- Then, convert short-term participation into medium-term treasury capital through redeem timing and share mechanics.

Our end goal is not just to run one successful game round.  
It is to solve a larger problem:  
**How an Agent Vault can bootstrap sustainable, iterative capital in its early stage.**

### What comes next

We have built the core FoMON3D gameplay and treasury framework.  
Next, we will continue shipping the Agent strategy execution module on Nad.fun (for example, initial buy + TWAP sell) and harden risk controls.

## 中文版

如果我们相信未来会有大量 Agent 参与经济活动，一个现实问题就摆在眼前：  
**Agent 可能有能力赚钱，但人类不一定愿意第一天就把钱交给它。**

FoMON3D 的想法很简单：  
我们不直接要求用户“信任 Agent”，而是先给用户一个高参与度、可理解的游戏入口，然后把这份注意力和流动性转化为 Agent Vault 的可运作资本。

### 为什么这个思路成立？

传统路径是“先有策略，再找资金”。  
但对 Agent 来说，顺序可能要反过来：  
先用产品机制聚集人和钱，再让策略证明自己。

FoMON3D 正在做的是：

1. 用户存入 `MON`，获得 `FoMON`（LST 形式的份额凭证）。  
2. 倒计时博弈驱动持续参与：有人继续存入就刷新，没人存入就结算。  
3. 最后存入者获得奖励。  
4. 用户通过 `redeem` 机制退出。  
5. 在 redeem 等待窗口内，资金留在 Agent 管理的 Treasury 中，用于策略执行和收益尝试。

这不是“纯赌博”，也不是“纯理财”，而是一种**从游戏到资本形成**的中间层机制。

### 核心创新：把“信任”变成可量化的资本流

在 Web3 里，最难的不是写一个策略，而是让人愿意把真金白银交给策略。  
FoMON3D 用游戏机制把这一步拆成两段：

- 先通过即时反馈、竞争和奖励机制建立参与意愿。  
- 再通过金库份额和锁定窗口，把短期参与沉淀为中期资本。

也就是说，FoMON3D 的目标不是赢一局游戏，而是建立一个更长期的问题解法：  
**如何让 Agent Vault 在早期就拥有可持续、可迭代的资本来源。**

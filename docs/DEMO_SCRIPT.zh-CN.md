# FoMON3D 演示脚本（中文）

建议时长：2 分 30 秒到 3 分钟  
建议结构：问题 -> 方案 -> 玩法 -> 架构 -> Monad 价值 -> 路线图

## 0. 开场（0:00 - 0:15）

画面：
- 展示项目标题 `FoMON3D`
- 副标题：`FOMO Game + Treasury Shares on Monad`

旁白：
“大家好，我们是 FoMON3D。我们做了一个运行在 Monad 上的链上游戏，把高频 FOMO 回合和金库份额机制结合在一起。”

## 1. 痛点与机会（0:15 - 0:35）

画面：
- 左侧关键词：`Pure Speculation`
- 右侧关键词：`Pure Yield`
- 中间关键词：`FoMON3D Hybrid`

旁白：
“很多链游要么偏投机，刺激高但留存差；要么偏理财，可持续但传播弱。FoMON3D 的目标是把两者融合，既有回合紧迫感，也有份额价值支撑。”

## 2. 核心玩法（0:35 - 1:15）

画面：
- 展示流程图图片：`a3d6cac1950450a1afc934f1ed918110.png`
- 逐步高亮 1 到 5 步

旁白：
“用户先存入 `$MON`，系统铸造 `$FoMON`。每次有效存入都会把倒计时重置为 30 秒。倒计时结束前最后一个存入者赢得本轮奖励。  
未获胜用户不会直接归零，而是可以继续持有 `$FoMON`，或者进入 7 天赎回队列，或在 DEX 有流动性时直接交易。”

## 3. 合约架构（1:15 - 1:50）

画面：
- 四模块框图：
- `GameRound`
- `FoMONToken`
- `TreasuryVault`
- `StrategyAdapter`

旁白：
“架构上我们做了四层拆分。`GameRound` 管回合与结算，`FoMONToken` 管份额代币，`TreasuryVault` 管资产和赎回队列，`StrategyAdapter` 负责策略执行和风险约束。”

## 4. 为什么是 Monad（1:50 - 2:15）

画面：
- 依次出现：`Low Latency`、`Low Fees`、`High Throughput`

旁白：
“FoMON3D 是高频实时交互场景，对链性能敏感。Monad 的低延迟、低费用和高吞吐可以让倒计时竞速体验更流畅，且支持小额高频参与。”

## 5. 收尾与路线图（2:15 - 2:45）

画面：
- 里程碑：
- `v0` 核心合约与模拟
- `v1` 测试网与前端

旁白：
“当前我们完成了机制和架构设计，下一步是合约实现、测试网部署和公开测试。FoMON3D 不是一次性玩法，而是可持续迭代的博弈加份额系统。谢谢大家。”
